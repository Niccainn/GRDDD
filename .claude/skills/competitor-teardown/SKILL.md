# Skill: competitor-teardown

## When to invoke

- "How do we compare to X" / category sizing
- Before building or revising a `/compare/<x>` page
- A repeated objection names an alternative
- A `marketing-feedback-loop` finding flags a competitive gap

## Discipline

Same rule as `docs/GRID-MARKET-STATE.md`: factual, no marketing words, no strawman. Steelman the competitor. A teardown that only flatters GRID is useless for actual positioning.

## Procedure

```
1. Define the comparison axis honestly: what job is the user hiring
   for? Compare on THAT, not on a feature checklist GRID happens to win.

2. For the competitor, capture (from their public surfaces only —
   site, docs, pricing; never speculation):
   - What they're genuinely good at (steelman — be specific)
   - Their actual pricing/packaging
   - Where they'd legitimately win this user

3. For GRID, capture (cross-checked against GRID-MARKET-STATE — no
   aspiration as fact):
   - Where GRID genuinely wins this user, and why (defensible)
   - Where GRID legitimately loses today (state it plainly)

4. Output: a factual table + a one-paragraph "honest take" — when to
   pick GRID, when to pick them. If a /compare page comes from this,
   the page must survive the competitor's own team reading it.

5. Log any "they do X we should consider" as a marketing-feedback-loop
   finding (innovation lane if it's a wedge, fix lane if it's a gap).
```

## Verification

- Competitor is steelmanned (a real strength stated specifically)
- GRID's genuine weaknesses are named, not buried
- Every GRID claim cross-checked vs `GRID-MARKET-STATE.md`
- No banned words; would not embarrass us if the competitor read it
- Actionable signals logged to the loop

## Failure modes

- **Strawmanning** — the fastest way to lose a sophisticated buyer who knows the competitor. Steelman or don't ship.
- **Feature-checklist framing** — "we have 47 integrations they have 12" ignores the job. Compare on the job.
- **Aspiration leak** — citing GRID roadmap as a present advantage. Market-state discipline applies.
- **Speculation** — inferring the competitor's internals/roadmap. Public surfaces only; label inference as inference.
